# Connect Zibur to OpenAI securely The included `index.html` works on GitHub Pages in contextual guidance mode. Open-ended OpenAI answers require a server-side endpoint because an API key must never be stored in browser HTML or JavaScript. ## Suggested deployment 1. Create a Vercel project. 2. Add `api/zibur.js` from this package. 3. Add the environment variable `OPENAI_API_KEY` in the hosting dashboard. 4. Set `APP_ORIGIN` to the public Beyond The Visa site origin. 5. Optionally set `OPENAI_MODEL`; the default is `gpt-5.6-terra`. 6. Deploy, then copy the public endpoint ending `/api/zibur`. 7. In the app open **Ask Zibur → AI connection**, paste that secure endpoint and save. The backend prompt prevents Zibur from claiming to diagnose patients or replace emergency, clinical, immigration, legal, tax or financial professionals. Current rules can be checked using OpenAI web search, but users should still follow the linked responsible authorities.
